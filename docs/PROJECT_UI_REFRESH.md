# 今日工作台界面重构

## 定位

本轮是 Phase 6 正式发布前的界面重构工作，不替代路线图中的 Phase 7“小团队可能性验证”。

目标是让计划、执行和复盘三个阶段采用适合各自任务的页面结构，同时保持现有数据模型、任务状态、SQLite 和备份格式不变。

## 已实现

### 计划阶段

- 不再展示文件夹或星系视图。
- 使用全宽四步任务创建流程：标题、四象限、重复规则、确认。
- 四象限只保留四张卡片作为唯一输入入口。
- 顶部步骤标签同步当前选择。
- 下方展示今日已安排任务、象限、重复和顺延标记。
- 顺延待确认任务置于计划流程之前。
- 任务模板按需展开。

### 执行阶段

- 文件夹或星系视图占满主要工作区。
- 底部工具坞提供新增任务、今日任务、搜索和设置。
- 同一时间只打开一个右侧抽屉。
- 切换抽屉不会清空尚未提交的任务标题或搜索条件。
- 当前主任务显示独立状态提示。

### 星系视图

- 四个象限使用低饱和透明色雾区分。
- 星球颜色与所在象限保持对应，完成后仍保留象限颜色。
- 图例同时提供文字，颜色不是唯一信息来源。
- 飞船沿现有曲线路径巡航。
- 系统启用“减少动态效果”时停止巡航和星球呼吸，显示静态飞船。

### 复盘阶段

- 当前文件夹或星系视图作为降亮背景。
- 未完成任务处理和四问复盘位于右侧抽屉。
- 四问逐项填写，返回后保留组件内草稿。

## 能力边界

- 不修改任务状态机和实际耗时计算。
- 不新增持久化字段。
- 不实现复盘草稿跨重启保存。
- 不修改月度总览。
- 不引入动画框架或大型前端依赖。
- 不改变 Phase 7 的团队验证范围。

## 自动测试边界

覆盖：

- 计划阶段隐藏主视图。
- 分步新增任务和象限唯一输入。
- 抽屉互斥及草稿保留。
- 星系四象限语义和曲线路径。
- 复盘背景与分步问题。
- 原有任务执行、模板、搜索、数据安全和月度能力回归。

## 手动验收

正式发布前检查：

1. 桌面窗口下计划内容无需围绕空白视图滚动。
2. 执行视图填满可用区域。
3. 四象限区域和星球颜色可快速区分。
4. 抽屉不会遮挡到无法操作当前任务。
5. 窄屏抽屉改为底部面板。
6. 中文长标题不会覆盖其他控件。
7. 减少动态效果开启后所有功能仍可使用。
