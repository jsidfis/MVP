# 二阶段方案：便携体验版

当前执行状态：便携体验版已完成基础打包和验证。对应实现计划见 `docs/superpowers/plans/2026-06-21-portable-preview-implementation.md`。

## 1. 阶段目标

二阶段第一步目标是把当前 MVP 从“开发环境可运行”推进到“可以发给其他用户体验”的便携版。

目标交付物是一个压缩包：

```text
每日计划与复盘-v0.2.0-portable.zip
```

用户解压后双击应用程序即可体验，不需要安装 Node.js、Rust、VSCode 或 Visual Studio；但目标电脑需要已有 Microsoft Edge WebView2 Runtime。

## 2. 核心假设

- 当前优先级是降低日常打开成本，而不是做账号、云同步或团队协作。
- 体验版应该可以直接发送给别人，不要求对方具备开发环境。
- 便携版的数据应该跟随解压文件夹移动，方便复制、备份和转发。
- 示例数据应该可修改，但不能污染用户自己的正式数据。
- 便携体验版先服务 Windows 用户。

## 3. 本阶段范围

本阶段包含：

- 便携包目录结构。
- 便携数据目录。
- 首次启动选择“空白开始 / 查看示例”。
- 正式数据和示例数据分离。
- 示例数据库初始化。
- 快速开始说明。
- 打包脚本或命令。
- 基础验收检查。

本阶段不包含：

- 账号登录。
- 云同步。
- 团队协作。
- 自动更新。
- 跨平台发布。
- 完整安装器体验。
- 复杂模板系统。

## 4. 便携包结构

建议便携包解压后结构如下：

```text
每日计划与复盘/
  每日计划与复盘.exe
  data/
    user.sqlite
    demo.sqlite
  docs/
    快速开始.txt
  assets/
    icon.ico
```

说明：

- `每日计划与复盘.exe` 是用户双击启动的入口。
- `data/user.sqlite` 保存用户正式数据。
- `data/demo.sqlite` 保存示例体验数据。
- `docs/快速开始.txt` 说明启动、备份、重置和转发方法。
- 具体文件名可根据 Tauri 实际产物调整，但用户入口必须清晰。

## 5. 数据策略

便携版数据存放在应用所在文件夹内，而不是系统 AppData。

数据规则：

- 空白开始使用 `data/user.sqlite`。
- 查看示例使用 `data/demo.sqlite`。
- 两个数据库互不影响。
- 删除 `data/user.sqlite` 可以重置正式数据。
- 删除 `data/demo.sqlite` 后，下次进入示例模式可以重新生成示例数据。
- 复制整个便携文件夹时，数据跟随文件夹一起移动。

风险说明：

- 用户删除整个便携文件夹时，数据也会一起删除。
- 因此快速开始文档必须明确提醒：备份时复制整个文件夹，或至少复制 `data/` 目录。

## 6. 首次启动流程

首次启动时，如果没有选择过工作模式，显示启动选择页：

1. 空白开始
   - 创建或打开 `data/user.sqlite`。
   - 进入今日工作台。

2. 查看示例
   - 创建或打开 `data/demo.sqlite`。
   - 如果示例库不存在，写入示例数据。
   - 进入今日工作台。

选择后可以在设置中切换：

- 我的数据
- 示例数据

切换时需要清楚提示当前使用的是哪个数据空间，避免用户误以为示例数据是自己的正式记录。

## 7. 示例数据设计

示例数据采用“真实个人一天的语境，但覆盖主要功能”。

示例内容应包含：

- 今日目标。
- 四象限任务。
- 已完成任务。
- 进行中任务。
- 未开始任务。
- 顺延任务和顺延原因。
- 完整晚间四问复盘。
- 少量历史日期，用于月度总览展示趋势。

示例任务风格：

- 工作：整理项目计划、回复关键消息、处理临时事项。
- 学习：阅读文档、整理笔记。
- 生活：运动、采购、休息。
- 复盘：总结完成情况和明日重点。

示例数据不应包含用户真实隐私信息。

## 8. 打包策略

优先使用现有 Windows GNU 工具链：

```powershell
npm run tauri:build:gnu
```

在此基础上增加便携包整理步骤：

1. 运行前端和 Tauri 构建。
2. 收集生成的 `.exe` 和必要运行文件。
3. 创建 `data/` 和 `docs/` 目录。
4. 写入快速开始说明。
5. 生成 zip 压缩包。

如果 Tauri 产物默认更接近安装器而不是便携目录，需要单独整理一个 release staging 目录，避免把构建缓存、源码或开发依赖放进 zip。

## 9. 快速开始说明内容

便携包内至少包含这些说明：

- 如何启动：双击应用程序。
- 数据在哪里：`data/` 目录。
- 如何备份：复制整个文件夹，或复制 `data/`。
- 如何重置正式数据：删除 `data/user.sqlite`。
- 如何重置示例数据：删除 `data/demo.sqlite`。
- 如何发给别人体验：压缩整个便携文件夹，不要只发送 `.exe`。
- 注意事项：不要把便携文件夹放到需要管理员权限的系统目录。

## 10. 验收标准

本阶段完成时必须满足：

- 从干净目录解压便携包后可以双击启动。
- 不安装 Node.js、Rust、VSCode 或 Visual Studio 也可以运行。
- 空白开始会创建正式数据库。
- 查看示例会创建示例数据库。
- 修改示例数据不会影响正式数据。
- 修改正式数据不会影响示例数据。
- 关闭并重新打开后数据仍然存在。
- 删除对应数据库后可以按规则重建。
- 包内说明能让新用户理解启动、备份、重置和转发。
- GitHub README 指向便携体验版使用方式。
