# 今日工作台界面重构设计

## 1. 背景

当前今日工作台在计划、执行和复盘三个阶段共用同一套双栏布局：

- 左侧长期展示文件夹或星系视图。
- 右侧纵向堆叠顺延确认、任务新增、模板、搜索和设置。

这套布局已经承载完整功能，但产生三个明显问题：

1. 星系视图只使用细分割线划分四象限，无法一眼识别任务的重要程度和紧急程度。
2. 计划阶段的录入内容较长，固定侧栏导致左右重心失衡；向下滚动后左侧视图空旷且失去作用。
3. 执行和复盘阶段的核心视图被常驻侧栏压缩，飞船、轨迹和任务星球缺少足够空间和层次。

本轮目标是在不改变业务规则和数据模型的前提下，让三个阶段拥有适合各自任务的页面结构。

## 2. 设计目标

- 计划阶段聚焦录入和安排，不展示无实际作用的文件夹或星系视图。
- 执行阶段让文件夹或星系视图占据主要窗口。
- 复盘阶段保留今日轨迹作为背景，同时提供专注的分步处理流程。
- 四象限无需依赖阅读任务标题即可快速辨认。
- 次要工具按需出现，不再长期占据固定侧栏。
- 动画提供趣味和状态反馈，但不承担数据语义。
- 保持现有任务开始、完成、计时、顺延、模板、重复、搜索和设置能力。

## 3. 非目标

本轮不包含：

- 月度总览改版。
- 数据模型或 SQLite schema 变更。
- JSON 备份和导入格式变更。
- 任务状态机、计时规则或顺延规则变更。
- 新增复杂重复任务规则。
- 账号、云同步、团队协作或移动端。
- 引入大型动画或画布渲染依赖。
- 用动画替代按钮、文本或任务状态提示。

## 4. 总体页面结构

`DailyWorkspace` 按当前阶段渲染三种页面骨架：

### 4.1 计划阶段

- 隐藏文件夹视图和星系视图。
- 使用全宽计划工作区。
- 顺延待确认任务位于新增任务流程之前。
- 新增任务使用分步卡片。
- 下方持续展示今日已安排任务摘要。
- 模板入口保留在计划阶段，通过按需面板打开。

### 4.2 执行阶段

- 文件夹或星系视图填满工作区可用区域。
- 保留主页视图切换并记住用户上次选择。
- 右侧浮动抽屉展示当前任务、实际计时和操作。
- 底部工具坞提供新增任务、今日任务、搜索和设置入口。
- 模板不作为执行阶段的常驻入口。

### 4.3 复盘阶段

- 继续显示用户选中的文件夹或星系视图。
- 背景视图降低亮度和对比度，保留已完成任务、旗帜和飞行轨迹。
- 右侧浮动抽屉分步处理未完成任务和晚间四问。
- 所有待处理内容完成后显示今日完成概览。

## 5. 计划阶段交互

### 5.1 新增任务步骤

新增任务按以下顺序进行：

1. 任务标题。
2. 四象限。
3. 重复规则。
4. 确认添加。

用户可以点击已经经过的步骤返回修改。未提交的输入在步骤切换和面板开关过程中保留。

### 5.2 四象限选择

四象限只有一个可操作输入入口：四张象限卡片。

- 重要且紧急。
- 重要不紧急。
- 不重要但紧急。
- 不重要不紧急。

顶部步骤标签只显示当前结果，例如 `四象限 · 重要且紧急`，不再提供第二套选择控件。任务摘要使用相同标签和颜色同步显示结果。

### 5.3 今日任务摘要

摘要至少显示：

- 任务标题。
- 四象限标签。
- 重复标记。
- 顺延标记。

摘要用于维持整体计划感，不复制文件夹或星系的完整交互。

## 6. 执行阶段交互

### 6.1 全屏工作区

工作区使用稳定的可视高度，视图填满标题栏和底部工具坞之间的空间。页面主体不再因固定右侧栏形成长距离纵向滚动。

文件夹和星系使用同一外层框架：

- 顶部保留日期、阶段切换、视图切换和月度总览。
- 中间是主视图。
- 底部是浮动工具坞。
- 右侧是按需抽屉。

### 6.2 工具坞与抽屉

工具坞提供：

- 新增任务。
- 今日任务。
- 搜索。
- 设置。

同一时间只打开一个抽屉。再次点击当前入口关闭抽屉；点击其他入口切换抽屉。

抽屉关闭不清空尚未提交的新任务或搜索条件。任务模板从计划阶段的按需入口打开，不在执行工具坞中重复提供。

### 6.3 当前任务抽屉

任务星球第一次点击后：

- 任务按照现有规则进入当前主任务状态。
- 飞船开始沿曲线路径巡航。
- 抽屉显示任务标题、开始时间和当前实际耗时。
- 文案明确提示再次点击星球会完成任务。

第二次点击当前星球后：

- 调用现有完成任务动作。
- 飞船抵达星球。
- 星球插旗。
- 轨迹转为完成状态。

不改变多任务和后台计时规则。

## 7. 星系视觉规则

### 7.1 四象限区域

四个象限使用低饱和、半透明的区域色雾：

- 重要且紧急：红色系。
- 重要不紧急：金黄色系。
- 不重要但紧急：青蓝色系。
- 不重要不紧急：灰绿色系。

色雾必须保留深色星空底图，不形成四块不透明色板。

### 7.2 星球颜色

星球使用所在象限的同色系，完成状态通过旗帜、完成轨迹和适度的状态变化表达，不能把所有完成星球统一改成同一种颜色而丢失象限信息。

星系提供紧凑图例。颜色不是唯一信息来源，星球的可访问名称和任务详情仍显示完整象限文字。

### 7.3 轨迹与动画

- 当日第一个任务仍从四象限中心出发。
- 后续轨迹仍从上一个完成星球连接到下一个任务星球。
- 轨迹沿用已有稳定曲线路径。
- 当前飞船沿该路径适度巡航，不使用快速、持续闪烁或大幅缩放。
- 星球和背景只做缓慢呼吸或漂移。
- 没有当前任务时，飞船做低强度待机动画。
- 插旗和抵达反馈应短暂，不能阻塞任务状态更新。

在 `prefers-reduced-motion: reduce` 下：

- 停止巡航、呼吸和背景漂移。
- 保留静态飞船位置、轨迹和任务状态。
- 所有操作保持可用。

## 8. 复盘阶段交互

### 8.1 背景

复盘背景沿用当前主页视图：

- 星系视图保留星球、旗帜和轨迹。
- 文件夹视图保留四层任务结构。
- 背景整体降亮，避免和表单争夺注意力。

### 8.2 分步抽屉

右侧抽屉按以下顺序处理：

1. 未完成任务的顺延、放弃或改期。
2. 对需要顺延的任务填写原因。
3. 晚间四问。
4. 今日状态。
5. 今日完成概览。

每次只突出一个处理项或问题，并显示清晰的步骤进度。现有复盘字段和提交规则保持不变。

### 8.3 草稿行为

在阶段内切换问题、关闭抽屉或查看背景时，未提交输入不得丢失。离开当天或重载应用前的持久化保护不在本轮扩大范围；如现有仓库未支持持久草稿，本轮只保证组件生命周期内保留。

## 9. 组件边界

建议按现有 React 结构进行小步拆分：

- `DailyWorkspace`：阶段路由和顶层布局。
- `PlanningWorkspace`：顺延确认、分步新增和今日摘要。
- `TaskCreationFlow`：新增任务步骤和临时表单状态。
- `ExecutionWorkspace`：全屏主视图、工具坞和抽屉。
- `WorkspaceDock`：按需工具入口。
- `WorkspaceDrawer`：互斥抽屉容器。
- `GalaxyView`：星系区域、星球、轨迹和飞船。
- `ReviewWorkspace`：降亮背景和分步复盘抽屉。

现有 `TaskQuickAdd`、`SearchPanel`、`TaskTemplatePanel` 和 `SettingsPanel` 优先复用内部业务与表单能力。只有在现有结构无法适配分步卡或抽屉时才拆分，不复制数据操作逻辑。

## 10. 数据流与状态

- 任务、设置和每日文件继续由 `useAppStore` 提供。
- 阶段切换继续使用现有 `Stage` 类型和每日文件状态。
- 主视图选择继续使用 `settings.homeView`。
- 抽屉开关和新增任务步骤属于页面临时 UI 状态，不写入 SQLite。
- 任务开始和完成继续调用 `startTask` 与 `completeTask`。
- 搜索、模板、设置、顺延和复盘继续调用现有 store 动作。

本轮不创建新的 repository 接口。

## 11. 响应式与可访问性

### 11.1 桌面

- 主视图占满可用窗口。
- 抽屉从右侧覆盖部分视图，不改变主视图布局宽度。
- 工具坞保持固定尺寸，动态内容不能推动主视图跳动。

### 11.2 窄屏

- 右侧抽屉改为底部面板。
- 面板高度受限，内部内容独立滚动。
- 星球点击区域保持可操作尺寸。
- 任务标题允许换行，不遮挡相邻星球或工具。

### 11.3 可访问性

- 所有抽屉入口使用按钮，并提供明确名称和按下状态。
- 打开抽屉后，焦点进入抽屉；关闭后返回触发按钮。
- Escape 关闭非阻塞抽屉。
- 四象限不能仅依赖颜色表达，必须同时提供文本。
- 动画遵循减少动态效果设置。

## 12. 测试方案

### 12.1 自动测试

至少覆盖：

- 计划阶段不渲染文件夹或星系主视图。
- 计划新增任务按步骤推进并可返回修改。
- 四象限只存在一套可操作选择，结果同步到步骤标签。
- 新任务提交仍传递标题、象限和重复规则。
- 执行阶段可以切换文件夹和星系视图。
- 工具坞抽屉互斥切换，关闭后输入状态保留。
- 星系星球包含象限样式或数据标记。
- 第一次点击星球开始任务，第二次点击当前星球完成任务。
- 复盘阶段渲染背景视图和复盘抽屉。
- 现有顺延、模板、搜索、设置和任务执行测试继续通过。

### 12.2 视觉与手动验收

在桌面和窄屏窗口检查：

1. 计划阶段主要内容在常见窗口内完成，不需要为了填写基础字段进行长距离滚动。
2. 执行阶段视图填满可用空间，常驻侧栏消失。
3. 不阅读任务标题时，可以通过区域和星球颜色区分四象限。
4. 抽屉不会遮挡当前任务到无法操作。
5. 飞船沿曲线运动，关闭动画后功能不变。
6. 复盘填写时背景信息可辨认但不抢焦点。
7. 文件夹视图在新外层框架下仍可开始和完成任务。
8. 中文长任务标题不会溢出按钮、星球标签或抽屉。

### 12.3 构建验证

实现完成后运行：

```powershell
npm.cmd run test:run
npm.cmd run build
npm.cmd run tauri:check:gnu
npm.cmd run portable:build:gnu
git diff --check
```

便携包需要完成一次启动 smoke，并检查 zip 中不包含源码和本地用户数据。

## 13. 验收标准

- 计划模式不显示文件夹或星系视图。
- 计划录入使用全宽分步卡，四象限只有一个输入入口。
- 执行和复盘视图不再被固定长侧栏压缩。
- 四象限区域色雾和星球颜色形成稳定对应。
- 执行阶段使用全屏视图、底部工具坞和右侧浮动抽屉。
- 复盘阶段使用降亮背景和右侧分步抽屉。
- 飞船采用适度巡航动画，并支持减少动态效果。
- 现有任务、计时、顺延、模板、重复、搜索、设置和备份能力不回归。
- 不新增大型依赖，不修改持久化格式。
