# 每日计划与复盘桌面应用设计方案

## 1. 背景与目标

本产品是一款个人使用的每日计划与复盘桌面应用。第一版目标是帮助用户完成一个稳定的每日闭环：

1. 早上明确今天要做什么。
2. 白天执行任务并允许随时新增任务。
3. 晚上复盘完成情况、处理未完成事项，并为明天做调整。

第一版按个人本地工具设计，后续可以开源到 GitHub。当前不做团队协作、账号登录、云同步或微信小程序。

## 2. 核心假设

- 用户主要是个人自用，需要低压力、低摩擦的计划与复盘工具。
- 任务记录不要求完全实时，允许事后修改时间和耗时。
- 趣味动画用于增强反馈和仪式感，不承担强制记录职责。
- 第一版应优先保证每日计划与复盘闭环可用，再扩展月度文件柜和飞行地图。
- 数据优先本地保存，未来再考虑导出、备份或同步。

## 3. MVP 范围

MVP 聚焦“今天工作台”，包括：

- 今日目标。
- 今日任务列表。
- 四象限任务分类。
- 文件夹视图与星系飞船视图切换。
- 任务开始、完成、暂停、后台进行中和耗时修正。
- 未完成任务的复盘处理。
- 每日状态记录。
- 晚间复盘四问。
- 今日完成概览。
- 早晚提醒时间设置。
- 轻量只读月度总览入口。

月度总览第一版只做基础只读视图，不做完整文件柜抽屉动画和完整飞行地图。

## 4. 产品流程

### 4.1 首次启动

首次启动时，应用引导用户设置两个提醒时间：

- 早上计划提醒。
- 晚上复盘提醒。

设置后再请求系统通知权限。用户之后可以在设置中修改或关闭提醒。

### 4.2 打开应用

应用默认进入“今天工作台”，并恢复用户上次选择的主页视图：

- 文件夹视图。
- 星系飞船视图。

系统根据时间自动建议当前阶段：

- 早上：计划。
- 白天：执行中。
- 晚上：复盘。

用户可以随时手动切换阶段。

### 4.3 早上计划

早上计划阶段先显示“待确认顺延任务”。这些任务来自前一天复盘时选择顺延到今天的事项。

顺延任务规则：

- 不直接混入今日任务。
- 先进入待确认列表。
- 用户确认后加入今日事项。
- 加入后保留“顺延”标记。
- 用户也可以放弃或改到其他日期。

新增任务的最小字段：

- 任务标题。
- 四象限分类。

新增入口：

- 顶部快速输入：输入标题后选择四象限。
- 区域内新增：在某一楼层或某一星系象限内新增时，自动带入该象限。

### 4.4 白天执行

用户可以点击任务开始执行。系统记录开始时间。

星系飞船交互：

- 点击任务星球，飞船开始飞向该星球，该任务成为当前主任务。
- 当日第一个任务从四象限中心点出发。
- 后续任务从上一个完成星球出发。
- 轨迹使用随机柔和曲线连接两个星球。
- 第二次点击当前星球时，飞船抵达并插旗，任务完成。
- 系统记录完成时间。

多任务进行规则：

- 允许多个任务处于进行中。
- 同一时间只有一个任务是飞船当前目标。
- 其余进行中任务为后台进行中。
- 点击另一个任务成为飞船目标时，系统询问：暂停当前任务，或继续后台计时。

时间规则：

- 开始时间和完成时间用于计算实际耗时。
- 用户可以直接修改开始时间、完成时间或最终耗时。
- 修改时间不要求填写说明。
- 动画只是趣味反馈，不强制用户实时记录。

### 4.5 晚上复盘

晚上复盘阶段逐个处理未完成任务。每个未完成任务必须选择一种处理方式：

- 顺延到明天。
- 放弃。
- 改到指定日期。

处理时先选择原因标签，再可选补充说明。

默认原因标签：

- 时间估计错误。
- 临时事务插入。
- 精力不足。
- 依赖他人或外部条件。
- 优先级变化。
- 任务定义不清。
- 主动放弃或不再需要。

复盘内容采用四问结构：

1. 今天完成了什么？
2. 未完成的原因是什么？
3. 今天状态或感受如何？
4. 明天的重点是什么？

未完成原因可以由任务处理时的标签自动汇总，用户也可以补充整体说明。

### 4.6 复盘完成反馈

复盘完成后显示今日完成概览，不做评价性打分。

概览包含：

- 今日完成数量。
- 重点完成情况。
- 实际耗时。
- 今日状态。
- 未完成处理结果。

重点完成情况展示加权完成率，但不命名为效率分。例如：

- 今日完成：5 / 8。
- 重点完成情况：2 / 3 个重点任务完成。
- 实际耗时：4h 20m。
- 今日状态：4 / 5，附一句状态备注。
- 未完成处理：2 个顺延，1 个放弃。

## 5. 四象限设计

任务采用四象限分类：

- 重要且紧急。
- 重要不紧急。
- 不重要但紧急。
- 不重要不紧急。

文件夹视图和星系视图使用同一套分类数据。

## 6. 文件夹视图

文件夹视图是“当日文件”，不是四个独立文件夹。它像一栋四层楼的备忘录待办：

- 4F：重要且紧急。
- 3F：重要不紧急。
- 2F：不重要但紧急。
- 1F：不重要不紧急。

每层使用不同颜色区分。任务仍然以备忘录待办形式展示。

同一楼层内任务按状态排序：

1. 进行中。
2. 未开始。
3. 已完成。
4. 顺延或放弃。

文件夹视图偏向快速编辑、计划和复盘。

## 7. 星系飞船视图

星系视图是同一批任务的趣味化执行地图。

视觉规则：

- 四象限固定分区。
- 任务显示为星球。
- 飞船代表当前主任务。
- 当日第一个任务从四象限中心点出发。
- 后续轨迹从上一个完成星球连接到下一个任务星球。
- 轨迹使用随机柔和曲线，不使用直线。
- 完成任务后在星球上插旗。
- 若没有选择下一个任务，飞船进入待机或度假动画。

星系视图偏向执行反馈和仪式感，不替代数据记录本身。

## 8. 月度总览

主页提供“月度总览”按钮。MVP 中月度总览为轻量只读入口。

第一版月度总览包含：

- 本月日期格子或节点。
- 每日是否有记录。
- 每日完成概览的轻量摘要。
- 点击某一天可以查看对应当日文件。

暂不做：

- 完整 30/31 格文件柜抽屉动画。
- 完整月度飞行地图。
- 月度复杂统计。

后续版本可以扩展为：

- 文件柜视图：一个月 30/31 个格子，每个格子抽出一份当日文件。
- 飞行地图视图：一个月 30/31 个节点，点击节点弹出对应当日任务。

## 9. 每日状态

每日状态保持极简：

- 整体状态 1-5 分。
- 一句话状态说明。

状态用于解释任务完成情况和复盘，不直接参与效率分或压力型评价。

## 10. 数据模型

### 10.1 DailyFile

表示某一天的当日文件。

字段：

- 日期。
- 当前阶段。
- 今日目标。
- 每日状态评分。
- 每日状态说明。
- 复盘四问内容。
- 今日完成概览。

### 10.2 Task

表示一条任务。

字段：

- 标题。
- 四象限分类。
- 状态。
- 是否顺延。
- 顺延来源日期。
- 顺延原因标签。
- 可选顺延说明。

### 10.3 TaskSession

表示一次任务计时或执行片段。

字段：

- 任务 ID。
- 开始时间。
- 完成时间。
- 是否手动修正。
- 最终耗时。

### 10.4 ReviewDecision

表示复盘时对未完成任务的处理。

字段：

- 任务 ID。
- 处理结果：顺延、放弃、改期。
- 目标日期。
- 原因标签。
- 可选说明。

### 10.5 UserSettings

表示用户本地设置。

字段：

- 上次主页视图。
- 早上计划提醒时间。
- 晚上复盘提醒时间。
- 通知开关。

## 11. 任务状态

MVP 使用以下状态：

- `not_started`：未开始。
- `active_primary`：当前主任务。
- `active_background`：后台进行中。
- `paused`：暂停。
- `completed`：已完成。
- `postponed`：已顺延。
- `dropped`：已放弃。

## 12. 技术方案

推荐技术栈：

- 桌面壳：Tauri。
- 前端：React + TypeScript。
- 本地数据：SQLite。
- 动画：前端 Canvas 或 Pixi.js。
- 通知：Tauri notification 插件。

技术边界：

- 不做服务端。
- 不做账号系统。
- 不做云同步。
- 不做团队协作。
- 不做移动端或微信小程序。

本地数据优先保存在应用数据目录中。未来可以增加 JSON 或 Markdown 导出、备份和同步。

## 13. MVP 验收标准

MVP 完成时应满足：

- 用户可以完成早上计划、白天执行、晚上复盘的完整日循环。
- 用户可以新增任务，最少只填标题和四象限。
- 用户可以在文件夹视图和星系视图之间切换，并记住上次选择。
- 文件夹视图能按四层楼展示任务，并按状态排序。
- 星系视图能展示四象限星球、飞船、曲线轨迹和完成插旗。
- 任务可以记录开始和完成时间，并允许直接修改耗时。
- 未完成任务在复盘时必须被处理为顺延、放弃或改期。
- 顺延到今天的任务进入待确认列表，确认后加入今日事项并保留顺延标记。
- 每日状态为 1-5 分加一句话。
- 复盘为四问结构。
- 复盘完成后显示今日完成概览。
- 本地数据关闭应用后仍保留。
- 首次启动可以设置早晚提醒时间。

## 14. MVP 暂不做范围

MVP 不做：

- 账号、登录、云同步、团队协作。
- 完整月度文件柜抽屉动画。
- 完整月度飞行地图。
- 自动重复任务。
- 常用任务列表。
- 任务模板系统。
- 自定义原因标签。
- 复杂统计报表。
- 周报或月报。
- 移动端。
- 微信小程序。
- 数据导入导出。

## 15. 后续版本方向

后续可以考虑：

- 完整月度文件柜和飞行地图。
- JSON / Markdown 导出。
- 本地备份与恢复。
- 常用任务或模板。
- 周报/月报。
- 多设备同步。
- 小团队共享。

这些方向不进入 MVP，避免第一版范围膨胀。
