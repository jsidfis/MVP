# Windows 启动与星系飞行改造设计

## 1. 背景

当前 Windows 便携版通过桌面快捷方式直接启动
`D:\daily plan\每日计划与复盘\daily-plan-review.exe`。程序会同时打开命令行窗口，
关闭该窗口后应用也会退出。

星系视图还存在两个视觉问题：

- SVG 轨迹和使用百分比定位的星球采用不同的缩放方式，宽屏下轨迹端点无法落在星球中心。
- 轨迹只按序号切换正反弧线，形态重复且生硬；当前飞行器是简单三角形。

## 2. 目标

- Windows 发布版双击后只显示应用窗口，不显示命令行窗口。
- 开发版继续保留终端，方便读取日志。
- 第一段轨迹从四象限中心连接第一颗任务星球。
- 后续轨迹准确连接前后两颗任务星球的中心。
- 每段轨迹具有不同但稳定的曲线，应用重启后不变化。
- 飞行器改为迷你探索型 UFO，并沿路径自动朝向前进方向。
- 减少动态效果时保留静态 UFO 和完整任务操作能力。

## 3. 非目标

- 不修改任务状态机、计时、顺延或数据模型。
- 不引入图片资源、Canvas、Three.js 或新依赖。
- 不改变桌面快捷方式路径。
- 不让随机轨迹在重渲染、切换视图或重启时跳动。

## 4. Windows 启动

在 Tauri 的 Windows 发布入口声明 GUI 子系统：

```rust
#![cfg_attr(not(debug_assertions), windows_subsystem = "windows")]
```

该声明只作用于非调试构建。`tauri:dev:gnu` 仍从终端运行并输出日志，
便携发布版则不创建控制台窗口。

重新生成便携包后，用新构建的 `daily-plan-review.exe` 替换 D 盘现有程序。
桌面快捷方式已经直接指向该文件，因此不需要重建。

## 5. 轨迹坐标与随机规则

### 5.1 坐标一致性

星球继续使用 `0..100` 的百分比坐标。轨迹 SVG 设置
`preserveAspectRatio="none"`，让 SVG 的横纵坐标分别映射到容器完整宽高，
与绝对定位的百分比星球保持一致。

### 5.2 稳定伪随机

每段轨迹以任务 ID 及起终点生成确定性种子。种子控制：

- 曲线弯向。
- 两个控制点在路径上的前后位置。
- 垂直于直线方向的偏移幅度。
- 两个控制点不完全相同的偏移量。

算法不使用 `Math.random()`。同一任务在相同起终点下始终得到相同路径，
不同任务通常得到不同曲线。偏移幅度按起终点距离限制，避免短路径过度弯曲。

## 6. UFO

UFO 使用内联 SVG 图形，包含：

- 圆润封闭座舱。
- 椭圆机身。
- 左右短翼。
- 底部推进光。

UFO 继续通过 SVG `animateMotion` 沿当前路径巡航，并使用 `rotate="auto"` 自动朝向。
静态降级图形使用同一 UFO 轮廓，不再使用 CSS 三角形。

## 7. 测试与验收

自动测试覆盖：

- 同一输入重复计算得到完全相同的路径。
- 不同任务 ID 得到不同的控制点。
- 路径字符串的起点和终点与路线坐标一致。
- 轨迹 SVG 使用 `preserveAspectRatio="none"`。
- 当前飞行器渲染 UFO 的座舱、机身和推进光。

构建验证：

```powershell
npm.cmd run test:run
npm.cmd run build
npm.cmd run tauri:check:gnu
npm.cmd run portable:build:gnu
git diff --check
```

手动验收：

1. 解压新便携包并双击应用，确认不出现命令行窗口。
2. 关闭应用窗口，确认进程正常结束。
3. 在宽屏星系视图检查每段轨迹准确连接星球中心。
4. 切换视图和重启应用，确认轨迹形状保持不变。
5. 检查 UFO 沿曲线运动且不遮挡主要任务信息。
